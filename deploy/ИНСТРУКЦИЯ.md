# Деплой КОНДА на Timeweb VPS — Полная инструкция

---

## ШАГ 1 — Создать сервер на Timeweb

1. Откройте сайт **timeweb.cloud** и войдите в личный кабинет
2. В меню слева нажмите **«Облачные серверы»**
3. Нажмите кнопку **«Создать сервер»**
4. Выберите настройки:
   - **Операционная система**: Ubuntu 22.04 LTS
   - **Тариф**: самый минимальный (1 CPU, 1–2 GB RAM вполне достаточно)
   - **Название**: напишите любое, например `conda`
5. Нажмите **«Создать»**
6. Подождите 1–2 минуты пока сервер запустится
7. На вашу почту придёт письмо с тремя данными — **сохраните их**:
   - IP-адрес сервера (например: `185.241.52.110`)
   - Логин: `root`
   - Пароль (длинный, автоматически сгенерированный)

---

## ШАГ 2 — Настроить DNS у регистратора домена

> Это нужно сделать ДО установки, чтобы SSL-сертификат настроился сразу.

Зайдите на сайт, где вы покупали домен (например: nic.ru, reg.ru, 2domains.ru и т.д.)

Найдите раздел **«DNS-записи»** или **«Управление DNS»** для вашего домена.

Добавьте **две записи типа A**:

| Тип записи | Имя (хост) | Значение (IP) | TTL |
|:---:|:---:|:---:|:---:|
| A | @ | ВАШ_IP_АДРЕС | 3600 |
| A | www | ВАШ_IP_АДРЕС |3600 |

> Где `@` — это сам домен (например `konda.ru`), а `www` — это `www.konda.ru`

Сохраните изменения. DNS обновляется обычно за 15–30 минут, иногда до 24 часов.

## ШАГ 3 — Подключиться к серверу по SSH

SSH — это способ управлять сервером через текстовые команды с вашего компьютера.

### Вариант А — через PowerShell (встроен в Windows, рекомендуется)

1. Нажмите **Win + R**, введите `powershell`, нажмите Enter
2. В открывшемся синем окне введите команду (замените IP на ваш):

```
ssh root@185.241.52.110
```

3. Нажмите **Enter**
4. Если появится вопрос вида:
   ```
   Are you sure you want to continue connecting (yes/no/[fingerprint])?
   ```
   Введите `yes` и нажмите Enter

5. Появится запрос пароля:
   ```
   root@185.241.52.110's password:
   ```
   Вставьте пароль из письма Timeweb (**Ctrl+V** или правой кнопкой мыши).
   
   > ⚠️ Когда вводите пароль — символы НЕ отображаются (это нормально). Просто вставьте и нажмите Enter.

6. Если всё верно — вы увидите приглашение командной строки:
   ```
   root@conda:~#
   ```
   Вы подключены к серверу!

### Вариант Б — через PuTTY (если PowerShell не работает)

1. Скачайте PuTTY: **putty.org** → Download → `putty-64bit-X.XX-installer.msi`
2. Установите и откройте PuTTY
3. В поле **Host Name** введите IP вашего сервера
4. **Port**: 22
5. Нажмите **Open**
6. На вопрос «Accept?» нажмите **Accept**
7. Логин: введите `root`, Enter
8. Пароль: вставьте из письма, Enter

## ШАГ 4 — Выполнить 3 команды на сервере

Теперь вы находитесь на сервере. Выполняйте команды **по одной** — скопируйте, вставьте, нажмите Enter, дождитесь завершения.

---

### Команда 1 — Скачать скрипт установки

Скопируйте и вставьте в SSH-окно:

```bash
curl -O https://raw.githubusercontent.com/ekbkondadom-ctrl/Proekt_Managers/main/deploy/setup.sh
```

Нажмите **Enter**. Вы увидите что-то вроде:
```
  % Total    % Received ...
100  3521  100  3521    0     0   ...
```
Это нормально — скрипт скачан.

---

### Команда 2 — Открыть скрипт и вписать ваш домен

```bash
nano setup.sh
```

Нажмите **Enter**. Откроется текстовый редактор прямо в терминале. Вы увидите содержимое скрипта.

**Что нужно сделать:**

Найдите строку (она в самом верху, 8-я строка):
```
DOMAIN="ВАШ_ДОМЕН.ru"
```

Нажимайте клавишу **→** (стрелка вправо) или используйте **стрелки ↑↓** чтобы добраться до этой строки.

Когда курсор стоит на `ВАШ_ДОМЕН.ru`:
- Выделите и удалите текст `ВАШ_ДОМЕН.ru` клавишей **Delete** или **Backspace**
- Напечатайте ваш реальный домен, например: `konda.ru`

Строка должна стать такой (с вашим доменом):
```
DOMAIN="konda.ru"
```

**Сохранение и выход:**
1. Нажмите **Ctrl + O** (буква O, не ноль) — редактор спросит имя файла
2. Нажмите **Enter** — файл сохранён
3. Нажмите **Ctrl + X** — редактор закрылся

Вы снова в командной строке.

---

### Команда 3 — Запустить установку

```bash
bash setup.sh
```

Нажмите **Enter**.

Скрипт начнёт работу. Вы будете видеть надписи о каждом шаге. Это займёт **3–7 минут**.

Ничего не нажимайте пока не появится вопрос:

```
Настраивать SSL прямо сейчас? (y/n):
```

Если DNS уже настроен (прошло 30+ минут после Шага 2) — введите `y` и нажмите Enter.

Если DNS ещё не распространился — введите `n`, Enter. SSL можно настроить позже.

## ШАГ 5 — Открыть сайт в браузере

Откройте браузер и перейдите на `http://ВАШ_ДОМЕН.ru`

При первом открытии появится форма создания **главного аккаунта (Супер-администратор)**:
- Придумайте логин (например: `admin`)
- Придумайте надёжный пароль (минимум 8 символов)
- Нажмите «Создать»

Всё готово — сайт работает!

---

## Если SSL (https) нужно настроить позже

Подключитесь к серверу по SSH и выполните:

```bash
certbot --nginx -d ВАШ_ДОМЕН.ru -d www.ВАШ_ДОМЕН.ru
```

## Обновление сайта в будущем

Когда вы захотите обновить сайт новой версией кода:

**На вашем компьютере:**
```
git push
```

**На сервере (по SSH):**
```bash
bash /var/www/conda/deploy/update.sh
```

---

## Полезные команды (на сервере)

| Что делает | Команда |
|---|---|
| Посмотреть статус | `pm2 status` |
| Посмотреть логи | `pm2 logs conda-backend` |
| Перезапустить сайт | `pm2 restart conda-backend` |
| Обновить сайт | `bash /var/www/conda/deploy/update.sh` |

```bash
pm2 logs conda-backend --lines 50   # Показать последние ошибки
nginx -t                            # Проверить настройки Nginx
systemctl status nginx              # Статус веб-сервера
```
